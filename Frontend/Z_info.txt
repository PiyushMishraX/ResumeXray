The React Context API setup involves three core steps: creating the context, providing the state, and consuming the value. 1.  Create Context: Use createContext() to define a context object.  2. Create Provider: Build a component that holds the state (using useState or useReducer) and renders <Context.Provider value={...}> to wrap child components. 3.  Consume Context: Use the useContext() hook in any descendant component to access the provided data. 

import React, { createContext, useState, useContext } from 'react';

// 1. Create the context
const ThemeContext = createContext();

// 2. Create the Provider component
export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');
  
  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

// Export the context for consumption
export default ThemeContext;

Infinite Render Loops / React Errors: Calling Maps() modifies the history state of the router. Modifying state during a render triggers a re-render. React will catch this and throw an error (usually something like "Cannot update a component while rendering a different component").
How useNavigate is meant to be used: The Maps function is an imperative hook. It is designed to be called inside event handlers (like a button click) or inside a useEffect hook, which runs after the render phase completes.
If you really wanted to use useNavigate here, you would have to wrap it in a useEffect

The <Navigate /> component is a declarative wrapper around useNavigate. Instead of telling React to change the page right now (imperative), you are telling React: "If there is no user, the UI for this component is a redirect" (declarative).
